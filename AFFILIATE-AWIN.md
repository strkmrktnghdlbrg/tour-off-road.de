## Empfohlene Reihenfolge der Integration
1. **HanseMerkur** -> Vergleichstabelle `/versicherungen/` (höchste Provision, bereits genannt). Allianz Direct als 2. Zeile.
2. **Bergzeit + Globetrotter + Fritz Berger** -> je Ausrüstungs-Detail ein „auch bei …"-Button neben Amazon.
3. **Reifen.com** -> beim KO2-Reifen-Eintrag + künftigen Reifen-/Recovery-Artikeln.
4. **Sunny Cars + Roadsurfer** -> Reiseplanungs-Hub (Schritt „vor Ort / Anreise") + Reiseziel-Hubs.
5. **Outdooractive** -> Routen-Detailseiten (Aside „Tour planen / GPX") und `/routen/`.
6. **Direct Ferries** -> Marokko-/Tunesien-Reiseziel-Hubs (Fähranreise). AWIN-Verfügbarkeit vorab prüfen, sonst eigenes Programm.

## Technische Umsetzung (IMPLEMENTIERT)
Die Deeplink-Infrastruktur ist gebaut und im Build verifiziert:

- **`src/site.ts`** — globale Config: `awin.publisherId` (leer bis Annahme), `amazon.partnerTag`, plus Helper `awinDeeplink({ merchantId, target, clickref })`. Baut `https://www.awin1.com/cread.php?awinmid=…&awinaffid=…&ued=<encoded>&clickref=…`. Fällt auf den rohen Direktlink zurück, solange `publisherId` leer ist -> Seite funktioniert auch VOR der Freigabe.
- **`src/products.ts`** — `awinMerchants`-Registry mit allen recherchierten `awinmid`s (HanseMerkur 11705, Bergzeit 12557, Fritz Berger 70949, Reifen.com 7605, Sunny Cars 13830, Roadsurfer 76432, Outdooractive 17147 …). Plus `categoryMerchant` (Default-Partner je Ausrüstungs-Kategorie), `merchantLink(key, {target, clickref})` und `gearAffiliateHref(data, clickref)`.
- **Verdrahtet:** Ausrüstungs-Listen (`/ausruestung/` + `/ausruestung/[category]/`) ziehen ihren Link über `gearAffiliateHref` (Kategorie-Default, per `merchant`-Feld im Frontmatter überschreibbar); HanseMerkur-Zeile in `/versicherungen/` über `merchantLink('hansemerkur', …)`.

### So gehst du nach der AWIN-Annahme vor
1. In **`src/site.ts`** `awin.publisherId` = deine Publisher-ID (`awinaffid`) eintragen. -> ab dann werden ALLE Links automatisch zu getrackten Deeplinks.
2. In **`src/products.ts`** je angenommenem Programm die `awinmid` prüfen/ergänzen (für Globetrotter & Direct Ferries steht sie noch auf `null` = Direktlink).
3. Optional pro Produkt im Frontmatter `merchant: "<key>"` setzen, um den Kategorie-Default zu überschreiben; `affiliateUrl` mit echter Produkt-Landingpage füllen (statt `#`), dann wird genau dorthin verlinkt.

Verifiziert: mit gesetzter Publisher-ID erzeugt der Build z.B.
`…/cread.php?awinmid=7605&awinaffid=<ID>&ued=https%3A%2F%2Fwww.reifen.com%2F&clickref=gear-bfgoodrich-ko2`.

Hinweis: Publisher-ID nicht ins Repo committen — entweder direkt in `site.ts` lokal setzen oder später über eine Env-Variable beziehen. `clickref` pro Platzierung wird automatisch aus dem Produkt-/Kontext-Slug gesetzt.
